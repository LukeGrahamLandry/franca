// Terminology (borrowed from Zig)
// 
// - Strictly checked illegal behaviour: Results in a panic in all optimisation modes. 
//     - `assert(false)`
//     - unwraping an empty optional
// - Safety checked illegal behaviour: Results in a panic in safe mode and UB in unchecked mode.  
//     - `sanity(false)`
//     - getting caught breaking your pinky promises.
//     - reading off the end of an array. 
// - Undefined behaviour: Mistakes the compiler cannot detect. Your program will behave unpredictably. 
//     - breaking your pinky promises.
//     - secret crimes.
//     - dereferencing a null raw pointer. 
//     - dereferencing an invalid raw pointer. 
//     - calling an ffi function that performs UB in the host language.
